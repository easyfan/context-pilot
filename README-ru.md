# context-pilot

**Протокол «безопасного забывания» для контекстного окна Claude Code.**

[English](README.md) | [中文](README-zh.md) | [Deutsch](README-de.md) | [Français](README-fr.md)

«`/clear`, а потом продолжай X» — инструкция, которую не может выполнить
ни одна модель: контекст, который её произносит, ею же и уничтожается.
context-pilot компилирует эту непроизносимую инструкцию в три шага на
уровне harness — записать самодостаточный handoff до очистки, дать
человеку нажать `/clear`, затем автоматически доставить handoff в холодную
сессию. Итог: длинная сессия пересекает очистку контекста **без потерь**,
вместо того чтобы тащить раздутый контекст (токен-налог на каждый ход) или
откатываться к auto-compact с потерями.

Плагин-близнец [quota-pilot](https://github.com/easyfan/quota-pilot):
quota-pilot паркует сессию через окно rate-limit (гибернация);
context-pilot переносит сессию через очистку контекста (амнезия с письмом
самому себе). Ключевое отличие: чекпоинт quota-pilot — страховка, handoff
context-pilot — **вся память** следующей сессии, поэтому его гейт обладает
правом вето.

## Компоненты

| Компонент | Назначение |
|---|---|
| **Skill** (`context-pilot`) | Протокол стороны записи: правило решения (работать дальше / clear / compact), аудит из шести компонент всего, что живёт только в разговоре, правила письма для «холодного читателя» и гейт самодостаточности с вето. |
| **Команда** (`/clear-then <следующий шаг>`) | Выполняет протокол с явно заданным следующим шагом и возвращает вам клавишу `/clear`. |
| **Хук SessionStart** (`context_deliver.sh`) | Доставка без привязки к source: после `/clear` (обычный Claude *или* приложение happy) свежий `context-handoff.md` инжектируется с преамбулой холодного читателя, затем потребляется (переименовывается). |
| **Хук PostToolUse** (`context_sample.sh`) | Сэмплинг: следит за заполнением контекста; выше 70 % (настраивается) инжектирует алерт оценки границы, не чаще одного раза за cooldown. |

## Правило решения

Три входа: **t** (оставшаяся работа), **H** (необходимый объём handoff),
**N** (расстояние до потолка — только срочность).

- **t мало** → просто доделайте; любой перенос — чистые накладные расходы.
- **Самодостаточность недостижима** → **не** очищать; доработать до
  настоящей границы или принять auto-compact.
- **Гейт пройден, H мало** → записать handoff, пригласить `/clear`.
- **Серая зона** → compact. Compact отказывает *мягко* (модель чувствует
  пробел и может перечитать); плохой clear отказывает *тихо* (новая сессия
  не знает, чего она не знает).

Handoff фиксирует: цель, конкретный следующий шаг, **решения, включая явно
отвергнутые альтернативы**, ловушки, честное непроверенное состояние,
устные ограничения пользователя — плюс карту сессии со ссылкой на старый
transcript (он переживает clear на диске). Указатели, никогда не
скопированное содержимое.

## Установка

Как плагин:

```
/plugin marketplace add easyfan/context-pilot
/plugin install context-pilot@context-pilot
```

Вручную:

```bash
git clone https://github.com/easyfan/context-pilot.git
cd context-pilot && ./install.sh          # --dry-run предпросмотр, --uninstall удаление
```

Ручная регистрация hook'ов (если `install.sh` прервался из-за отсутствия python3): добавьте следующее в `~/.claude/settings.json`:

```json
"hooks": {
  "SessionStart": [
    { "hooks": [ { "type": "command", "command": "$HOME/.claude/context-pilot/hooks/context_deliver.sh" } ] }
  ],
  "PostToolUse": [
    { "matcher": "*", "hooks": [ { "type": "command", "command": "$HOME/.claude/context-pilot/hooks/context_sample.sh" } ] }
  ]
}
```

## Конфигурация

Необязательный `~/.claude/context-pilot/config.json`:

```json
{
  "context_window": 200000,
  "warn_pct": 70,
  "cooldown_minutes": 15,
  "check_seconds": 60
}
```

> **Примечание для моделей с окном 1M:** значение `context_window` по умолчанию — `200000`. Если у вашей модели контекстное окно в 1M токенов (например, идентификаторы моделей с `[1m]`), установите `1000000` — со значением по умолчанию оповещения срабатывают слишком рано и могут показывать использование выше 100%.

Окно свежести доставки: `CONTEXT_PILOT_FRESH_SECONDS`, по умолчанию 900.
